# SpringBoot Data JPA Project

This project is a Spring Boot application demonstrating the use of Spring Data JPA with MySQL. It includes a complete REST API for performing CRUD operations and derived queries on an `Employee` entity.

## Features

- **Spring Data JPA integration:** Seamlessly maps the Java `Employee` entity to a relational database table.
- **RESTful Endpoints:** Complete set of endpoints (GET, POST, PUT, DELETE) for managing employees.
- **Derived Query Methods:** Demonstrates advanced repository capabilities without writing raw SQL.
- **JPQL & Native Queries:** Examples of using `@Query` for both HQL and standard SQL queries.
- **Database Auto-generation:** Entity changes are automatically synced to the database schema via Hibernate.

---

## 🏗️ Architecture Layer

1. **Controller Layer**: Exposes the REST API logic at `/api/employees`.
2. **Service Layer**: Handles business logic and intermediate data validation.
3. **Repository Layer**: Interface inheriting from `JpaRepository` for DB access.
4. **Entity Layer**: Plain Java object representing the relational table mapping.

---

## 🛠️ Requirements & Setup

1. **Java Development Kit (JDK) 21** or later is required to run the application.
2. **MySQL Database:** Local MySQL server running on port `3306`.
3. **Maven:** Project dependency management.

### Database Setup

1. Open your MySQL client.
2. Connect with your credentials (by default, `root` / `root`).
3. Run the following command to create the required database:
```sql
CREATE DATABASE jpa_example_db;
```
*(The `cg_employee` table will be auto-generated by Hibernate when the application starts.)*

---

## 🚀 Running the Application

Ensure that your `application.properties` database user and password match your local environment. 

To start the application from your terminal:
```bash
./mvnw spring-boot:run
```
*(Use `.\mvnw` on Windows)*

The application will start on `http://localhost:8080`.

## 🧪 Testing with Postman

Import or manually enter the following requests into Postman/Insomnia:

### Create an Employee
- **Method:** `POST`
- **URL:** `http://localhost:8080/api/employees`
- **Body (JSON):**
  ```json
  {
      "name": "Jane Doe",
      "department": "Engineering",
      "salary": 75000.0
  }
  ```

### Get All Employees
- **Method:** `GET`
- **URL:** `http://localhost:8080/api/employees`

### Get Employee By ID
- **Method:** `GET`
- **URL:** `http://localhost:8080/api/employees/1`

### Update an Employee
- **Method:** `PUT`
- **URL:** `http://localhost:8080/api/employees/1`
- **Body (JSON):**
  ```json
  {
      "name": "Jane Smith",
      "department": "Management",
      "salary": 85000.0
  }
  ```

### Derived Queries Tests (GET)
- **By Department:** `http://localhost:8080/api/employees/department/Engineering`
- **By Name:** `http://localhost:8080/api/employees/name/Jane Doe`
- **By Name Containing:** `http://localhost:8080/api/employees/name/contains/Jane`
- **Salary Greater Than:** `http://localhost:8080/api/employees/salary/greater/50000`
- **Department and Salary Filter:** `http://localhost:8080/api/employees/filter?department=Engineering&salary=60000`
- **By Department (JPQL):** `http://localhost:8080/api/employees/jpql/department/Engineering`
- **Low Salary (Native Query):** `http://localhost:8080/api/employees/native/salary/low/40000`
